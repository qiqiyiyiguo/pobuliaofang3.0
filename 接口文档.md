# 题目列表接口文档

## 1. 接口概述
本组接口用于获取不同技术模块（Java、Web、Python、测试）的题目列表，支持分页和按类别筛选。前端通过 GET 请求调用，需携带认证 Token（通过 `Authorization` 请求头传递）。

---

## 2. 公共说明

- **基础地址**：`https://localhost:8080`（可在配置中修改）
- **请求头**：
  - `Content-Type: application/json;charset=utf-8`
  - `Authorization: Bearer {token}`（需登录后从 localStorage 获取）
- **响应格式**：所有接口统一返回 JSON 对象，包含 `code`、`message` 和 `data` 字段。
  - `code`：业务状态码，`200` 表示成功，其他表示失败。
  - `message`：提示信息，失败时描述错误原因。
  - `data`：成功时返回的具体数据，结构根据接口定义。

---

## 3. 接口列表

### 3.1 获取 Java 题目列表
- **URL**：`/question/Java`
- **方法**：GET
- **描述**：分页获取 Java 模块的题目，可按类别筛选。

#### 请求参数（Query）

| 参数名   | 类型   | 必填 | 默认值 | 说明                                                         |
| -------- | ------ | ---- | ------ | ------------------------------------------------------------ |
| page     | int    | 否   | 1      | 当前页码                                                     |
| pageSize | int    | 否   | 10     | 每页条数                                                     |
| category | string | 否   | 无     | 题目类别，可选值：`技术知识`、`项目经历`、`场景题`、`行为题`。不传则返回所有类别。 |

#### 请求示例
```
GET /question/Java?page=1&pageSize=10&category=技术知识
```

#### 成功响应示例（HTTP 200）
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "total": 35,
    "list": [
      {
        "id": 101,
        "title": "Java 垃圾回收机制",
        "category": "技术知识",
        "content": "简述 Java 的垃圾回收算法..."
      },
      {
        "id": 102,
        "title": "项目中使用过的设计模式",
        "category": "项目经历",
        "content": "请举例说明在项目中应用的设计模式..."
      }
    ]
  }
}
```

#### 错误响应示例
```json
{
  "code": 500,
  "message": "服务器内部错误"
}
```

## 4. 公共错误码说明

| 错误码 | 含义                     | 处理建议                       |
| ------ | ------------------------ | ------------------------------ |
| 401    | 未授权，Token 无效或缺失 | 前端应跳转登录页重新获取 Token |
| 404    | 请求的资源不存在         | 检查 URL 是否正确              |
| 500    | 服务器内部错误           | 联系后端排查日志               |
| 其他   | 业务自定义错误           | 根据返回的 `message` 提示用户  |

---

## 5. 注意事项
- 所有接口均需携带有效的 Token，否则返回 401。
- 分页参数 `page` 从 1 开始计数。
- 类别筛选字段 `category` 的值必须与后端数据一致，目前支持上述四种分类。
- 返回的 `list` 数组中，每个题目对象至少包含 `id`、`title`、`category`、`content` 字段，可根据实际需要扩展其他字段（如创建时间、难度等）。

# 用户接口文档

## 1. 接口概述
本组接口用于用户认证与个人资料维护，包含登录、注册、获取/更新用户信息、修改密码、退出登录。前端位于 `vue-project/src/api/user.js`，通过 `Authorization: Bearer {token}` 传递认证信息（登录/注册接口除外）。

---

## 2. 公共说明
- **基础地址**：`https://localhost:8080`（与题库接口一致，见 `vue-project/src/utils/request.js`）
- **请求头**：
  - `Content-Type: application/json;charset=utf-8`
  - `Authorization: Bearer {token}`（登录成功后前端通常存到 `localStorage token`，后续请求自动携带）
- **响应格式**：统一返回 JSON：`{ code, message, data }`
  - `code=200` 表示成功；非 200 表示业务失败
  - `message` 为提示信息
  - `data` 为业务数据（不同接口不同）

---

## 3. 接口列表

### 3.1 用户登录
- **URL**：`/api/user/login`
- **方法**：POST
- **描述**：使用用户名和密码登录，成功后返回 token 与用户信息（具体字段以后端为准）。

#### 请求参数（Body / JSON）
| 参数名   | 类型   | 必填 | 说明   |
| -------- | ------ | ---- | ------ |
| username | string | 是   | 用户名 |
| password | string | 是   | 密码   |

#### 请求示例
```json
{
  "username": "test",
  "password": "123456"
}
```

#### 成功响应示例（HTTP 200）
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "token": "Bearer eyJhbGciOi...",
    "user": {
      "id": 1,
      "username": "test",
      "nickname": "测试用户",
      "email": "test@example.com"
    }
  }
}
```

---

### 3.2 用户注册
- **URL**：`/api/user/register`
- **方法**：POST
- **描述**：注册新用户。

#### 请求参数（Body / JSON）
| 参数名   | 类型   | 必填 | 说明                                   |
| -------- | ------ | ---- | -------------------------------------- |
| username | string | 否   | 用户名（字段是否必填以后端校验为准）   |
| password | string | 否   | 密码（字段是否必填以后端校验为准）     |
| email    | string | 否   | 邮箱（字段是否必填以后端校验为准）     |
| 其他字段 | any    | 否   | 例如手机号、验证码等（按后端实际扩展） |

#### 请求示例
```json
{
  "username": "test",
  "password": "123456",
  "email": "test@example.com"
}
```

#### 成功响应示例（HTTP 200）
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 1
  }
}
```

---

### 3.3 获取当前用户信息
- **URL**：`/api/user/info`
- **方法**：GET
- **描述**：获取当前登录用户资料。

#### 请求参数
无

#### 成功响应示例（HTTP 200）
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 1,
    "username": "test",
    "nickname": "测试用户",
    "avatar": "https://example.com/avatar.png",
    "email": "test@example.com"
  }
}
```

---

### 3.4 更新用户信息
- **URL**：`/api/user/update`
- **方法**：PUT
- **描述**：更新用户资料（按后端允许更新的字段为准）。

#### 请求参数（Body / JSON）
| 参数名    | 类型   | 必填 | 说明               |
| --------- | ------ | ---- | ------------------ |
| nickname  | string | 否   | 昵称               |
| avatar    | string | 否   | 头像 URL           |
| email     | string | 否   | 邮箱               |
| 其他字段  | any    | 否   | 后端允许更新的字段 |

#### 请求示例
```json
{
  "nickname": "新的昵称",
  "avatar": "https://example.com/avatar.png"
}
```

#### 成功响应示例（HTTP 200）
```json
{
  "code": 200,
  "message": "success",
  "data": true
}
```

---

### 3.5 修改密码
- **URL**：`/api/user/change-password`
- **方法**：POST
- **描述**：修改登录用户密码。

#### 请求参数（Body / JSON）
| 参数名       | 类型   | 必填 | 说明     |
| ------------ | ------ | ---- | -------- |
| oldPassword  | string | 是   | 旧密码   |
| newPassword  | string | 是   | 新密码   |

#### 请求示例
```json
{
  "oldPassword": "123456",
  "newPassword": "654321"
}
```

#### 成功响应示例（HTTP 200）
```json
{
  "code": 200,
  "message": "success",
  "data": true
}
```

---

### 3.6 退出登录
- **URL**：`/api/user/logout`
- **方法**：POST
- **描述**：退出登录（如后端不维护会话，也可仅前端清 token；此接口用于可选后端登出）。

#### 请求参数
无

#### 成功响应示例（HTTP 200）
```json
{
  "code": 200,
  "message": "success",
  "data": true
}
```

## 4. 公共错误码说明
| 错误码 | 含义                     | 处理建议                       |
| ------ | ------------------------ | ------------------------------ |
| 400    | 参数错误                 | 检查请求参数格式与必填字段     |
| 401    | 未授权，Token 无效或缺失 | 前端应跳转登录页重新获取 Token |
| 403    | 无权限                   | 检查用户权限/角色              |
| 404    | 请求的资源不存在         | 检查 URL 是否正确              |
| 500    | 服务器内部错误           | 联系后端排查日志               |
